Что такое структуры данных?

Структуры данных – это структуры кода для хранения и организации связанных данных.
В Python существуют четыре встроенных структуры данных: список, кортеж, словарь и множество.

Зачем они нужны?
Структуры данных упрощают изменение, навигацию и доступ к информации. Структуры данных определяют способ сбора данных,
функциональные возможности, которые мы можем реализовать, и отношения между данными.

Что такое Hash Map?
HashMap — основан на хэш-таблицах, реализует интерфейс Map (что подразумевает хранение данных в виде пар ключ/значение).
Ключи и значения могут быть любых типов, в том числе и null
Хеш-таблицей называется структура данных, реализующая интерфейс ассоциативного массива (абстрактная модель «ключ –
значение»), которая обеспечивает очень быструю вставку и поиск: независимо от количества элементов вставка
и поиск (а иногда и удаление) выполняются за время, близкое к константе – O(1). По сути, это обычный массив, где
местоположение элемента зависит от значения самого элемента.

Что такое Linked List?
Linked List - это, структура данных, которая состоит из элементов, зовущихся узлами. В узлах хранятся данные, а между
собой узлы соединены связями. Связь — это ссылка на следующий или предыдущий элемент списка. Таким образом, узел
состоит из данных и одной/двух связей, а сам связный список — из узлов.

В каких случаях использовать Linked List и Hash Map?
HashMap полезен для коллекции на основе хеширования общего назначения, а Hashmap полезен для управления хаотичным
порядком элементов.
